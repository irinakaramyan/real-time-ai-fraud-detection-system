Requirements
Functional Requirements
1. The system shall allow users to submit transaction details.
2. The system shall analyze transaction data.
3. The system shall calculate a risk score for each transaction.
4. The system shall classify transactions as low, medium, or high risk.
5. The system shall flag suspicious transactions.
6. The system shall store transaction data in the database.
7. The system shall allow analysts to review flagged transactions.
8. The system shall allow administrators to manage fraud detection rules.

Non-Functional Requirements
1. The system should provide results quickly.
2. The system should store data securely.
3. The system should be easy to use.
4. The system should be scalable for future growth.
5. The system should be maintainable and well-structured.
6. The system should ensure reliability and availability.
